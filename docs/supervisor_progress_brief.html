Nesy-Gen EACL Revision Research Brief

A discussion-ready summary of the graph-verified radiology report generation revision: reviewer concerns, implemented evidence, open risks, key terms, and next milestones.

IU-Xray pipeline active Real PrimeKG integrated BLIP training in progress MIMIC-CXR pending full run

Executive Summary

The revision has moved from a manuscript-level proposal to a reproducible package with real PrimeKG integration, IU-Xray and MIMIC-CXR data pipelines, graph reasoning, sensitivity analysis, retrieval baselines, factuality evaluation, and qualitative visualization scripts.

The graph-verification side is now credible and testable. The main remaining scientific risk is generation quality: the first Swin-tiny plus DistilGPT2 model is weak, retrieval is factually stronger, and the current BLIP/stronger-model run will determine the next paper framing.

Quick Glossary

These short definitions are included so the discussion can stay focused on the research decisions, not on decoding metric names.

PrimeKG A large biomedical knowledge graph containing diseases, phenotypes, anatomy, drugs, genes, pathways, and relations between them. Here it is used to check whether report entities are medically connected in a plausible graph.
Radiology PrimeKG cache A smaller, reusable subset of PrimeKG built from entities that appear in IU-Xray or MIMIC-CXR. It avoids loading the full multi-gigabyte graph for every notebook run.
Entity linking The step that maps phrases in a radiology report, such as "pleural effusion" or "cardiomegaly", to PrimeKG node IDs. If this step is noisy, later graph reasoning can also become noisy.
Graph score A summary of how well the report's clinical concepts agree with PrimeKG. It combines several graph checks into one overall signal.
Finding-to-diagnosis score Measures whether extracted findings and disease/diagnosis-like entities are connected through PrimeKG. A high value suggests the report entities are graph-compatible.
Located-in/type score Checks whether linked entities have plausible biomedical types and anatomy-style grounding. This is why a high value is encouraging but not by itself sufficient for report quality.
Bio-temporal score Measures whether the induced graph paths satisfy the biological and temporal assumptions in the method. This is currently the weakest score, so it should be refined or described carefully.
Entity overlap A factuality check that compares the clinical concepts in a generated report with the concepts in the reference report.
Unsupported entities Clinical concepts that appear in a generated report but are not supported by the reference report. This is useful, but it is not the same as radiologist review.
BLEU / ROUGE Standard text-overlap scores. They help measure wording similarity, but they do not fully capture whether a radiology report is clinically correct.

IU-Xray Results So Far

The IU-Xray graph-verification pipeline is now running on real reports and real PrimeKG. These results are not yet the final generated-report results; they tell us whether the clinical concepts extracted from reports are coherent with the biomedical graph.

Part of the Pipeline What We Know How To Talk About It
Graph reasoning Working on IU-Xray with the radiology PrimeKG cache. The graph side is now operational and reproducible.
Finding-to-diagnosis consistency Looks promising. Extracted findings are often medically connected to diagnosis-like concepts in PrimeKG.
Located-in/type consistency Looks strong. The linked entities usually have plausible biomedical/anatomical types.
Bio-temporal consistency Needs more care. This component should be refined or framed carefully because it is the weakest graph signal.
Sensitivity analysis Implemented. We can now show how errors in entity extraction affect graph verification.

Generation and Factuality Status

The first neural generator is not strong enough to carry the paper. Retrieval is currently more factually reliable, and the BLIP run is testing whether a stronger pretrained image-language model can improve the generation side.

Risks To Be Honest About

  • The initial neural generator is not competitive enough for main claims.
  • The current hallucination metric is entity-level proxy evidence, not radiologist review.
  • The bio-temporal graph component needs more care.
  • MIMIC-CXR is supported in code but still needs full execution.
  • GitHub push of the latest cache resolver needs valid authentication.
